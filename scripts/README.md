# Guide: Creating and Using the `scripts/` Directory

This walkthrough shows you, step by step, how to create the shared `scripts/` folder and check that it exists so the assistant can find your files every time.

## 1. เปิดเทอร์มินัลในโปรเจ็กต์นี้
0. (ถ้ายังไม่เห็นหน้าต่างตัวแก้ไข) บนหน้าต่างแชตของ ChatGPT ให้คลิกปุ่ม **Open Workspace** หรือ **View Code** ที่มุมขวาบนก่อน ระบบจะเปิดแท็บโค้ดในเบราว์เซอร์ให้
1. ภายในแท็บโค้ด มองที่แถบเมนูด้านบนของตัวแก้ไข (บริเวณเดียวกับที่มีคำว่า **File / Edit / View**)
   - คลิกเมนู **Terminal** แล้วเลือก **New Terminal**
   - หน้าต่างเทอร์มินัลจะเปิดขึ้นที่แผงล่างของหน้าจอ (หากไม่เห็น ให้กด `Ctrl+`` หรือคลิกไอคอนรูปสี่เหลี่ยมที่มุมล่างขวาเพื่อเปิด/ปิดแผง)
   - ถ้ามีแท็บเทอร์มินัลอยู่แล้ว ให้คลิกเลือกแท็บนั้นแทนการเปิดใหม่
2. เมื่อมีเทอร์มินัลแล้ว ถ้าอยู่คนละโฟลเดอร์ ให้รันคำสั่งนี้เพื่อลงไปยังโปรเจ็กต์ที่กำลังทำงานอยู่:
   ```bash
   cd /workspace/sin12358
   ```
   - เมื่อรันเสร็จแล้ว พิมพ์ `pwd` เพื่อตรวจสอบว่าเราอยู่ในโฟลเดอร์ `/workspace/sin12358` จริง ๆ

## 2. สร้างโฟลเดอร์ `scripts/`
1. ใช้คำสั่งสร้างโฟลเดอร์ (หากยังไม่มี):
   ```bash
   mkdir -p scripts
   ```
   - แฟล็ก `-p` ช่วยให้คำสั่งไม่แจ้งเตือนหากโฟลเดอร์ถูกสร้างไว้แล้ว
2. พิมพ์ `ls` เพื่อตรวจสอบไฟล์/โฟลเดอร์ในปัจจุบัน คุณควรเห็น `scripts/` ปรากฏอยู่ (หากยังไม่เห็น ลองตรวจว่ากำลังอยู่ใน `/workspace/sin12358` ด้วยคำสั่ง `pwd`)

## 3. (ทางเลือก) สร้างโฟลเดอร์ย่อยภายใน `scripts/`
หากต้องการจัดหมวดหมู่ไฟล์เพิ่มเติม ให้สร้างโฟลเดอร์ย่อยตามประเภทไฟล์หรืองาน เช่น
```bash
mkdir -p scripts/python
mkdir -p scripts/sql
```

## 4. เพิ่มไฟล์ของคุณลงใน `scripts/`
1. ถ้าคุณมีไฟล์อยู่แล้ว ให้ย้ายหรือคัดลอกเข้าไปใน `scripts/` เช่น
   ```bash
   mv /path/to/my_script.py scripts/
   ```
2. หรือสร้างไฟล์ใหม่โดยตรง เช่น
   ```bash
   touch scripts/example.py
   ```

## 5. ตรวจสอบรายการไฟล์อีกครั้ง
หลังจากเพิ่มไฟล์แล้ว ให้ใช้คำสั่งต่อไปนี้เพื่อตรวจสอบว่าไฟล์อยู่ในที่ที่ต้องการ:
```bash
ls -R scripts
```
คุณจะเห็นโครงสร้างโฟลเดอร์และไฟล์ภายใน `scripts/`

## 6. แจ้งผู้ช่วยเมื่อพร้อมให้ตรวจดูไฟล์
เมื่อคุณต้องการให้ฉันช่วยตรวจหรือแก้ไขไฟล์ไหน เพียงบอก path ที่ต้องการ เช่น
- `scripts/example.py`
- `scripts/sql/import_customers.sql`
ฉันจะสามารถเปิดดูไฟล์นั้นได้ทันทีโดยไม่ต้องให้คุณวางโค้ดในแชทอีกรอบ

---
> หากต้องการเพิ่มคำอธิบายเพิ่มเติมของแต่ละไฟล์ ลองสร้างไฟล์ `scripts/INDEX.md` หรือ `scripts/README.md` (ไฟล์นี้) เพื่อจดบันทึกว่ามีไฟล์อะไรบ้าง และใช้อัปเดตทุกครั้งที่คุณเพิ่ม/แก้ไขไฟล์ใหม่
